# UBIG Semi Project 🐾

## 📖 프로젝트 개요
**UBIG**은 유기동물 입양, 봉사 및 펀딩을 연결하여 유기동물과 새로운 가족을 이어주는 종합 플랫폼입니다. 
동물 보호와 복지에 관심이 있는 사람들이 모여, 안전하고 체계적인 동물 입양 절차를 밟고 봉사활동 및 후원에 참여할 수 있는 커뮤니티 기능을 제공합니다.

## 🎯 주요 기능
- **회원 및 관리자 기능 (`Member` & `Admin`)**
  - 일반 사용자, 관리자 등급에 따른 권한 분리 (Spring Security 적용)
  - 마이페이지를 통한 입양/봉사/후원 내역 조회
  - 관리자 대시보드를 통한 사이트 전반의 컨텐츠 및 회원 관리

- **반려동물 입양 (`Adoption`)**
  - 유기동물 프로필 리스트업 및 상세 정보 확인
  - 입양 신청 및 심사/승인 프로세스

- **봉사활동 (`Volunteer`)**
  - 동물 보호소나 기관의 봉사 프로그램 모집 공고
  - 봉사 신청 및 스케줄 관리

- **후원/펀딩 (`Funding`)**
  - 유기동물 치료비나 보호소 운영 등을 위한 크라우드 펀딩
  - 펀딩 참여 결제 및 진행률 확인

- **커뮤니티 (`Community`)**
  - 입양 후기 작성 및 정보 공유
  - 반려동물 양육 팁 및 자유게시판 운영

## 🚀 프로젝트 실행 방법 (Docker 활용)
이 프로젝트는 **Docker** 환경이 모두 세팅되어 있어 간편하게 실행할 수 있습니다.

1. **사전 준비**: 로컬 환경에 `Docker Desktop` 설치 및 실행
2. **프로젝트 클론 및 폴더 이동**
   ```bash
   git clone [레포지토리 주소]
   cd 2026-bootcamp_semi_project
   ```
3. **Docker Compose를 통한 컨테이너 빌드 및 백그라운드 실행**
   ```bash
   docker-compose up -d --build
   ```
4. **접속**
   - 브라우저에서 `http://localhost:8080` 으로 접속하여 메인 화면 확인.

## 💻 담당 개발 기능 및 화면 (My Contributions)
### 1️⃣ 유기동물 입양 신청 전체 프로세스
- **기능 설명**: 입양 희망자가 유기동물의 상세 정보를 확인하고, 입양 신청 폼을 작성해 제출하는 클라이언트 지향 워크플로우를 구현했습니다.
- **👉 [입양 컨트롤러 코드 보러가기 (클릭)🍿](./UBIGSemiProject/src/main/java/com/ubig/app/adoption/AdoptionController.java)**

<details>
<summary><b>🛠 트러블 슈팅: 대량의 폼 데이터 유효성 검증 및 무결성 확보 (클릭하여 펼쳐보기)</b></summary>
<div markdown="1">

- **이전 문제점 (Problem)**: 입양 신청서는 질문 항목이 많아 사용자가 필수 항목을 누락하거나 잘못된 형식의 데이터를 보내는 경우가 잦았고, 이 비정상적인 데이터가 DB에 그대로 들어가거나 500 서버 에러를 발생시켰습니다.
- **어떻게 고쳤는가 (Solution)**: 프론트엔드(JavaScript 정규식)에서 1차 검증을 마치더라도 개발자 도구를 통한 API 강제 변조를 막기 위해 **서버 사이드 방어 로직을 추가**했습니다. Spring의 `@Valid`와 커스텀 검증 로직을 구현하여 조건 불충족 시 에러 메시지를 띄우고 이전 데이터가 유지된 채로 뷰(View)를 반환하도록 설계하여 시스템의 셧다운과 쓰레기 데이터 삽입을 완전히 차단했습니다.

</div>
</details>

### 2️⃣ 유저 마이페이지: 내 입양 신청 내역 조회 및 상태 조작
- **기능 설명**: 사용자가 본인의 '마이페이지'에서 작성한 입양 신청서의 실시간 진행 상태(대기 / 승인 / 반려)를 조회하고, 신청서를 수정하거나 취소(삭제)할 수 있는 기능을 전담했습니다.
- **👉 [마이페이지 조회 쿼리(Mapper XML) 보러가기](./UBIGSemiProject/src/main/resources/mappers/member-mapper.xml)**

<details>
<summary><b>💡 트러블 슈팅: 비정상적인 상태 조작(API 변조) 방지 (클릭하여 펼쳐보기)</b></summary>
<div markdown="1">

- **이전 문제점 (Problem)**: 사용자가 브라우저 개발자 도구를 이용하거나 HTTP 요청을 강제로 변조하면, 이미 관리자가 심사를 끝내 **'승인' 처리된 입양글마저 강제로 삭제/수정할 수 있는 심각한 보안 취약점**이 발견되었습니다.
- **어떻게 고쳤는가 (Solution)**: 브라우저에서 '수정/삭제' 버튼을 숨기는 퍼블리싱 수준에 그치지 않고, **백엔드(Controller/Service) 레벨의 검증을 도입**했습니다. 수정이나 삭제 API가 호출될 때 무조건 원본 DB의 현재 상태(Status) 값을 단건으로 조회해, `심사 대기중(WAIT)`인 상태가 아닐 경우 즉각 예외(Exception)를 반환하도록 비즈니스 로직을 견고하게 수정했습니다.

</div>
</details>

### 3️⃣ 관리자 전용 대시보드: 입양 신청서 검토 및 승인/반려 시스템
- **기능 설명**: 관리자가 전체 유저 접수 건을 리스트 형태로 한눈에 파악하고, 각 신청서를 상세 검토한 뒤 상태(승인, 반려, 삭제)를 업데이트할 수 있는 관리자 시스템을 구축했습니다.
- **👉 [관리자 페이지 처리 코드 보러가기](./UBIGSemiProject/src/main/java/com/ubig/app/admin/AdminController.java)**

<details>
<summary><b>🔥 성능 최적화: N+1 쿼리 문제 해결 및 트랜잭션 보장 (클릭하여 펼쳐보기)</b></summary>
<div markdown="1">

- **이전 문제점 (Problem)**: 관리자 페이지 접속 시 입양 신청서 1개를 조회할 때마다 작성자 정보(Member)와 동물 정보(Pet)를 가져오느라 반복적인 SELECT 쿼리가 발생하는 **N+1 문제로 인해 페이지 로딩이 느려지는 현상**이 있었습니다. 또한, 입양 승인과 동물 정보 업데이트 간 동기화가 이루어지지 않는 데이터 정합성 이슈가 있었습니다.
- **어떻게 고쳤는가 (Solution)**: 
  1. **MyBatis `ResultMap`과 다중 JOIN 기법**을 활용하여 수많은 SELECT 호출을 단 1회의 쿼리로 묶어 처리함으로써 로딩 병목을 최적화했습니다.
  2. 입양 신청서가 '승인' 상태로 넘어갈 때, 동물의 입양상태 코드도 강제로 '입양 완료'로 바뀌는 로직을 하나의 `@Transactional`로 묶어, 중간 서버 다운 시에도 한쪽 데이터만 바뀌는 참사를 방지했습니다.

</div>
</details>
